# Ecosystem Map

## Camadas

```
foundation -> infrastructure -> platform -> products
        \-> ecosystem (build-system, standards, templates, DX)
        \-> docs (architecture, learning, monetization)
        \-> research (papers, benchmarks, experiments)
```

## Relacoes principais

- `ecosystem/build-system` fornece base de compilacao para `foundation/*`
- `ecosystem/standards` define contratos para todos os modulos
- `foundation/*` fornece primitives para `infrastructure/*`
- `infrastructure/*` habilita `platform/*`
- `platform/*` acelera entrega em `products/*`

## Fluxo de evolucao

1. Build-system e standards
2. libs fundamentais (libft, libds, libalgo, libparser)
3. runtime/vm/networking
4. ai infra e dados
5. produtos monetizaveis

## Pontos de monetizacao

- Tooling de build e templates para times internos
- Bibliotecas de performance/open source support
- AI infra components as managed services
- Produtos SaaS sobre a platform
